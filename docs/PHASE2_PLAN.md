# XunDuTerminal Phase 2 实施计划

> 2026-07-17 方向调整：任务中心及其工作台模板、批量命令、SSH 隧道、通知界面和异常退出恢复快照均已按产品反馈移除，避免增加使用复杂度和后台状态写入。文件与 RDP 继续使用内联进度反馈，正常工作台持久化保持不变。下方原方案仅作为历史设计记录，不再代表当前界面。

## 目标

第二阶段历史方案聚焦“稳定恢复 + 日常运维生产力”，后续已根据实际使用反馈进行减法调整。

## 防回归原则

1. 新功能使用独立模块和版本化数据，不继续扩大核心组件的职责。
2. 所有持久化内容先校验再读取，写入采用临时文件 + 原子替换；密码、私钥口令不进入快照、模板、历史或导出。
3. 原生能力先提供沙箱模拟和失败状态，再接 UI；任何单项失败都不得阻塞工作台渲染。
4. 批量执行、传输和隧道均有并发上限、超时、取消和明确的终态，不做无限自动重试。
5. 每个里程碑必须通过 TypeScript 构建、Lint、Rust 单测和浏览器沙箱回归后才能进入下一个里程碑。

## 2.1 恢复与工作台模板（历史，已撤销）

### 实现

- 保存最近 5 份无凭据工作台快照，单份限制 4 MiB。
- 使用会话标记识别异常退出；启动后显示恢复提示，不自动覆盖用户当前状态。
- 工作台模板支持保存、应用为新工作台、重命名、删除、导入和导出。
- 应用模板时重新生成工作台、组件和布局节点 ID，避免会话与布局冲突。
- 导入文件使用版本号和结构校验，非法文件不写入本地存储。

### 验收

- 模拟损坏主存储后可以从最后有效快照恢复。
- 模板中不包含 SSH/RDP 密码，重复应用不会产生重复 ID。
- 取消恢复、删除模板和非法导入不改变当前工作台。

## 2.2 统一传输中心

### 实现

- SSH/本地下载和 RDP 上传发布到同一传输事件总线。
- 展示来源、目标、协议、进度、速度、开始时间、终态和错误原因。
- 支持运行中取消、失败任务重试、清理已完成记录；最多持久化 100 条历史。
- SSH 单文件下载支持从临时文件续传；目录任务按文件边界重试。RDP 剪贴板协议不支持续传时明确标注为重新传输。

### 验收

- 切换工作台不丢失正在传输的状态。
- 重复粘贴不会产生重复任务；取消和重试不会复用已失效的原生句柄。
- 进度必须单调，速度不得显示负数或非有限值。

## 2.3 批量执行与 SSH 隧道

### 批量执行

- 从服务器列表多选目标，执行前展示命令和目标预览。
- 并发范围 1-8，默认 3；单机超时范围 5-600 秒。
- 危险命令由规则检测并要求二次确认。
- 每台服务器独立显示排队、运行、成功、失败、超时和取消状态，可仅重试失败项。
- 结果最多保留 50 次任务，每台输出限制 1 MiB。

### SSH 隧道

- 支持本地、远程和动态转发；保存配置但不自动启动。
- 启动前校验地址、端口冲突和认证方式，运行时展示 PID、启动时间和状态。
- 关闭应用或删除配置时终止对应子进程；启动失败必须回收句柄。
- 密码认证无法安全地交给系统 OpenSSH 时拒绝启动并给出改用密钥/Agent 的提示，不将密码拼入命令行。

### 验收

- 批量任务取消后不再启动排队目标，失败重试只影响失败项。
- 隧道端口占用、认证失败和进程意外退出均能回到明确终态。
- 任何批量或隧道操作不得重启已有交互式终端。

## 2.4 通知与压力验证

### 实现

- 长任务完成、批量失败、传输失败和隧道掉线触发应用内通知。
- 系统通知为可选设置；同一任务终态只通知一次。
- 提供任务历史清理和通知静默开关。

### 压力矩阵

| 场景 | 基线 | 通过条件 |
| --- | --- | --- |
| 多终端 | 12 个终端持续 30 分钟 | UI 可交互，无意外断线 |
| 大输出 | 单终端 100 MiB 流式输出 | 内存稳定，输入无明显阻塞 |
| 文件目录 | 10,000 项目录 | 可滚动、排序和切换工作台 |
| 传输 | 4 个并发任务 + 取消/重试 | 状态终结一致，无重复任务 |
| 批量执行 | 20 台模拟服务器，并发 1/3/8 | 并发上限有效，结果完整 |
| 隧道 | 端口占用、掉线、手动停止 | 无残留子进程 |

## 2026-07-17 执行结果

### 已完成（历史实施结果）

- 2.1 至 2.4 的功能实现已落地，任务中心、恢复快照、工作台模板、统一传输、批量执行、SSH 隧道和通知均已接入。
- Lint 通过且无第二阶段新增警告；现存 14 条均为主应用既有警告。
- 浏览器沙箱端到端回归通过，覆盖第二阶段关键路径及现有终端、文件、RDP 和工作台交互。
- 压力回归通过：100 MiB 终端输出、10,000 项目录虚拟列表、150 条传输写入、进度单调性和 100 条历史上限。
- TypeScript/Vite 生产构建通过；Rust 单元测试 24/24 通过。
- Tauri Release 构建通过，成功生成主程序、MSI 和 NSIS 安装包；新主程序启动后保持响应。

### 简化调整

- 移除任务中心导航入口、界面模块和工作台模板状态，保留正常工作台持久化能力。
- 移除异常退出检测、恢复提示、定时快照、原生快照命令和旧快照数据；正常工作台保存不受影响。
- 顶部搜索重构为独立 SSH/RDP 浮层，查询状态不再进入应用根组件，也不再推开侧栏或触发工作台重排；支持分词匹配、快速访问、方向键选择、回车打开、Esc 关闭和一键清除。
- 命令面板改为按当前显示语言搜索，取消逐条入场动画和高成本模糊层。
- 侧栏开合不再对全部工作台窗格执行 FLIP 动画，避免终端密集布局下出现卡顿。
- 增加中英文布局溢出、侧栏响应、命令面板搜索、搜索期间根组件零重绘及 2,500 条连接搜索压力回归。

### 公开发布前保留验证

- 在真实 SSH/RDP 环境执行 12 终端 30 分钟长稳测试、4 路真实传输和 20 台真实/预发布服务器批量任务。
- 在真实 OpenSSH 环境复核应用异常退出后的隧道子进程清理；当前自动化已覆盖参数、状态和模拟生命周期。
- 暂不提升版本号，不创建公开 Release。
